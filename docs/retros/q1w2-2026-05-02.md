## What Didn't Go Well / Friction Points

### 1. Cross-package Clause aggregation gap (Story 2.1 design oversight)

Winston (Architect): Story 2.1 shipped a Clause registry that **cannot render anything outside `internal/protocol`**. `make spec-coverage` runs a single `go test` invocation in `internal/protocol/`; that binary loads only `init()` blocks from that package's test files. Story 2.2's Q5 Clauses register from `internal/server/integration_test.go` — they fire and are compile-time-checked, but **never appear in `docs/spec-coverage.txt`**. Stories 2.3 and 2.4 inherit the same gap. The audit story (W1's strategic moat) is half-built; downstream stories silently register Clauses into a void from the renderer's perspective.

**Captured:** `docs/deferred-work.md` "Cross-package Clause aggregation" section with three approach options. Approach (c) — per-package tables concatenated at render — is the most idiomatic.

**Counterfactual:** A party-mode design review of Story 2.1 (W1 action item 5, skipped) might have caught this. Circumstantial evidence that action 5 had real value even though it didn't bite W2's sequencing.

### 2. AC text and test-name tables are load-bearing under pressure

John (PM): Story 2.4's auditor flagged AC3 PARTIAL because the integration test asserted "no `elicitation/create` substring on stdout" but didn't pin the white-box pending-map state. The implementer correctly added `internal/server/bidi_internal_test.go` to discharge the literal three-property assertion. Lesson: **AC text and the named-test list are contract**, not aspiration. Don't loosen under "we know what we meant" pressure.

### 3. Test-hygiene defers are converging into one sweep

Amelia (Developer): Stories 2.3 and 2.4 deferred 7+ test-hygiene items spanning the same surface — bidi tests in `internal/server/server_test.go` and `internal/server/synctest_test.go`. Patterns:

- `time.Sleep`-based timing → channel-sync replacement (caught and fixed in 2.3)
- `_ = stdinW.Write(...)` swallowed errors (fixed in 2.3, flagged again in 2.4)
- `stdoutW` pipe never explicitly closed (2.3 + 2.4 defers)
- `bidirHandshake` omits `protocolVersion` field (2.4 defer; pattern shared across all bidi tests)
- Goroutine leaks via `done` channel never read on `t.Fatalf` paths
- No timeout/deadline on bidi happy-path tests

Touching them per-story would diverge sampling and elicitation siblings. Bundle into a single test-hygiene PR.

### 4. W1 action item 3 reversed by execution

Mary (Business Analyst): W1 retro committed to **NOT** hand-author `sprint-status.yaml`. The named revisit trigger ("if `bmad-create-story` / `bmad-sprint-status` are re-engaged and need machine-readable input") fired during W2 — `bmad-sprint-planning` machine-generated `sprint-status.yaml` on 2026-05-01. The intent was preserved (no hand-authored YAML) but the artifact now exists. Worth a footnote on the W1 retro action ledger; not a problem.

## Key Insights

### A. The audit story is the load-bearing dividend

Every W2 story consumed W1 audit-story infrastructure (Q60 goldens, AI9 gate, Clause registry, capability enum). The sequencing-and-discipline investment from W1 is what made W2 a 1-day push. Reaffirms W1 retro Insight A: **discipline > features**.

### B. AC-discovery shrinkage is a planning *feature*, not a miss

Three of four W2 stories shrank under the surface. The dev-notes discipline ("read existing code first") converted that from a problem into the planning model. Don't react by tightening AC pre-flight checks — react by **continuing to write dev-notes that anticipate verification work**.

### C. Cross-package boundaries are real Go constraints, not workflow accidents

The Clause aggregation gap exists because Go test binaries don't cross-import `_test.go` files. This is structural, not a design slip. Future audit-story extensions need to plan for this from the outset — either accept per-package render tables or pay the test-harness aggregation cost up front.

### D. The auditor caught what the implementer would have missed

Story 2.4 AC3 PARTIAL is the canonical example. The implementer wrote a wire-side test ("no bytes written"); the auditor demanded the white-box assertion ("pending-map empty + outboundIDCounter untouched"). Both are correct; only together do they discharge AC3 literally. **The triad's value is independence, not redundancy.**
